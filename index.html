⚠︎ Routing & loading — applies to every task
  • Every task must pass a unique task id (and the icon + display name) as a route parameter from the Tasks tab into the task screen. The task screen reads these from route.params on mount.
  • The task screen does not wait on a blank screen for backend data. The first thing it renders is the loading state: the task's icon and its name from the route params, so the user immediately sees which task they entered.
  • Only after real data arrives from the backend does the screen transition out of loading into the task's actual content. If the backend errors, loading still resolves and the error is surfaced — never a frozen spinner.

› 💓 Heart Rate Check Camera-based finger PPG · 60s measurement · task 2 of 7 🪙 +15 pts
Key facts — read first
  • User-facing name is "Heart Rate Check" — NEVER "HRV Measurement". "HRV" means nothing to users and makes them anxious when they google it.
  • Raw numbers translated to labels. Backend measures RMSSD, SDNN, stress index, R-R intervals — user only sees BPM + "Recovery" label (HRV translation) + "Calm Level" label (stress inverted, more fill = better).
  • 60-second measurement, single screen that transforms. Measurement screen and results screen are the same page — no transitions. Metric cards loading dots → fill with results. Ring: purple (filling) → green (done).
  • No guided breathing during measurement. Paced breathing inflates RMSSD by ~40–45% (Laborde et al., 2021), compromising clinical comparisons. Breathing exercise is the NEXT task.
  • Mini camera preview stays top-right throughout measurement. User needs to confirm their finger is still on the camera (Welltory pattern).

Translation — what user sees vs what backend stores

Designer: these are the only numbers/labels that ever appear on screen. Everything else in the raw PPG pipeline stays on the server.

Shown to user
  • Averaged BPM (number)
  • Recovery: "Great" / "Good" / "Normal" / "A bit low" / "Needs care" + bar
  • Calm Level: "Relaxed" / "Moderate" / "Tense" + bar
  • Body-state sentence ("Your body is calm today.")
  • Trend sparkline (Day 3+)
Stored on backend only
  • RMSSD (ms), SDNN (ms), pNN50
  • Stress index (0–100)
  • Raw R-R intervals
  • Signal quality timeline
  • Device model, measurement duration

All raw data feeds into the RRS (Relapse Risk Score · 10% biometrics weight), the clinician CRM, the home-screen Recovery card, and AI chat personalisation.

  1. Chest is ALWAYS subordinate to Core Exercise. Chest EV must stay below Core's +50 BC fixed reward. Side dish, never the entree.

  3. Never two jackpots in a row. If yesterday rolled any jackpot (Lucky, Epic, Legendary, or milestone), today rolls from Normal distribution only. Server-enforced.

Purpose & Position

Position 5 of 9. Motivational carrot positioned immediately after Core Exercise — the user sees the chest glowing on the path during the hardest task of the day (forward-pull motivation).

Clinical basis
  • Schultz 2016 (reward prediction error): Variable rewards with low-anchored expectation produce dopaminergic responses stronger than fixed rewards of equal average value.
  • Variable-ratio reinforcement: Sustains behavior longer than fixed-ratio. Chest = variable, Core Exercise = fixed. Combination makes Core feel dependable AND Chest feel alive.
  • Habit loop (Duhigg 2012, Clear 2018): Cue (Core complete) → Routine (open chest) → Variable Reward (BC amount).

The WOW mechanism — 3 ingredients

WOW fires when three ingredients line up simultaneously on the scratched-open card. If any one is missing, it's decoration. The delivery logic exists to force all three to line up as often as possible — ideally every single day.

1. Synchronicity
"This is exactly what I needed to hear right now."
Produced by signal-matched selection. Signal sources: today's check-in (mood, cravings, confidence, HALT, accountability), today's Core Exercise category, recovery phase, day number, streak, primary addiction, day of week. If the selection is random, synchronicity is impossible.
2. Borrowed authority they respect
"Wait, Matthew Perry said that? He went through this."
Produced by source tier. Voice credibility > prose quality. A B-tier sentence from a recovery memoirist beats an A-tier sentence from an anonymous Instagram poster every time.
3. Permission they couldn't give themselves
"I'm allowed to feel this. I'm allowed to be tired. I'm allowed to start over."
Produced by function tag. Permission is the most under-used function in motivational content. Most quotes tell users to do more. The rare quote that gives them permission is the one that creates the biggest emotional release.

Governing rules — non-negotiable content guardrails

These rules sit above every decision. When logic conflicts with these rules, the rules win.

  1. The golden rule. Every feature must satisfy BOTH engagement AND therapeutic value. A quote that only entertains fails. A quote that only teaches fails. The library is the intersection, not the union.
  2. Never validate negative self-beliefs.
    ❌ "I am an addict." — identity-sickness framing
    ❌ "You are only as sick as your secrets." — validates sickness as identity
    ❌ "You are broken, and that's okay." — broken-identity validation
    ✅ "Between stimulus and response there is a space..." — agency, not identity-sickness
    ✅ "Recovery is a practice, not a verdict." — practice framing, no diagnostic label
  3. Never show scary/negative clinical metrics. No quote that implies "relapse risk HIGH." No quote that frames the user as severe, chronic, or doomed. Positive framing only.
  4. The 2AM test. Before any quote ships, read it at 2am imagining yourself in a panic attack or post-slip shame spiral. Would it make you feel WORSE? If yes, kill it. Classic failures:
    ❌ "Every storm runs out of rain." — dismissive in crisis
    ❌ "What doesn't kill you makes you stronger." — gaslights actual trauma
    ❌ "Just breathe." — invalidating when breathing isn't the problem
  5. Middle East / global cultural safety. Ships in Riyadh, Tel Aviv, and the broader Middle East. No pork, alcohol, or gambling references as metaphors. No religious text verses as universal wisdom. No gendered pronouns where neutral works. No US-centric idioms ("knock it out of the park," "Hail Mary"). No culturally specific holidays or figures as shorthand.
  6. Tone of voice — Option B (warm / coachy / factual).
    ❌ Saccharine affirmations ("You are enough, you are loved, you are perfect.")
    ❌ Poetic body-talk ("Your body is a temple of light.")
    ❌ Mystical lyricism that doesn't convert to action
    Lean hard on pragmatic, clear, grounded voices. Lean away from mystical-poetic ones — even beloved ones like Rumi — unless the specific quote is pragmatic and action-oriented.
  7. Zero-typing. The quote node has no typing. Scratch, tap, save, add to widget, done.

7-layer selection algorithm
  1. Layer 1 — Category match to today's Core Exercise (CBT, Refusal Skills, Crisis Planning, Values, etc.).
  2. Layer 2 — Tone match to morning mood. Rough/Low → gentle/comfort only. Okay → any. Good/Great → preference for firm/stoic/athletic/challenge.
  3. Layer 3 — No repeat within 60 days. Hard filter. Non-negotiable.
  4. Layer 4 — Favorites get 0.5× weight. Let them collect new ones.
  5. Layer 5 — Function match to signals (filtered by Family × Function legality).
  6. Layer 6 — Author diversity. No author twice within 7 days.
  7. Layer 7 — Novelty injection. Every 14th day, force a never-shown-before quote — variable ratio, creates curiosity.

Level-up ceremony coordination

The identity level-up ceremony (8-second fade-to-dim overlay from Gamification) fires as a separate full-screen overlay AFTER Node 8's [Done] tap and BEFORE Node 9's pre-tap state appears.

Sequence on a level-up day
  1. User taps [Done] on Node 8 (Daily Close)
  2. 300ms crossfade → Level-up ceremony overlay begins (8 seconds: dim → level name → identity statement → celebration burst → social proof)
  3. Ceremony ends → 300ms crossfade → Node 9 pre-tap state appears ("All done. Seal the day.")
  4. User taps COMPLETE → standard Node 9 celebration fires with level-up text in Line 1

Why this order: the ceremony is the DEEP moment ("I'm The Resilient — I've been tested. I held."). Node 9 is the CELEBRATORY moment. Ceremony first = emotional weight. Node 9 second = dopamine reward. Putting the ceremony inside Node 9 would create a 16+ second screen that overstays.

Light tier / minimum path
